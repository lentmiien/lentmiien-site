1. POST http://192.168.0.21:3000/api/audio/upload
For uploading an audio recording and create a transcription job. Return the job id to the user.

```

  -F "audio=@/tmp/manual_api_test.wav;type=audio/wav" \
  -F "device_id=room-pi-01" \
  -F "sample_rate=16000" \
  -F "channels=1" \

  http://192.168.0.21:3000/api/audio/upload
```

```
{
  "job_id": "some-job-id"
}
```

For fetching the job status. Return that status, transcripted text (if done), and an output audio file id (optional)

Sample command on the user side:
```
curl -v \
  http://192.168.0.21:3000/api/audio/jobs/JOB_ID_HERE
```

Expect response to be a JSON object with `status`, `transcribed text`, and `output file id`, where only `status` is required.

3. GET  http://192.168.0.21:3000/api/audio/output/{output_audio_id}
For fetching the output audio file. Return the audio file data.

Sample command on the user side:
```
curl -L \
  -o /tmp/manual_api_output.wav \
  http://192.168.0.21:3000/api/audio/output/OUTPUT_AUDIO_ID_HERE
```
