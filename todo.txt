Yes — there are several good public/free-ish sources, and for **Japan specifically** the anchor source is **JMA / 気象庁**. My ears perk up a bit here, because Japan’s official data is excellent, but not always “nice modern JSON REST API” shaped. A lot of the most reliable official stuff is **Atom/XML feeds** rather than polished developer APIs.

## Short answer: best stack for Japan

If I were building a Japan-focused alert/dashboard app, I’d start with this combo:

1. **JMA 防災情報XML feed** — official Japan source for weather warnings, typhoons, earthquakes, tsunami, volcano, flood/landslide/tornado info, etc. Public pull-style Atom/XML, no user registration, but you must handle polling/caching politely. ([data.jma.go.jp](https://www.data.jma.go.jp/developer/))  
2. **P2Pquake JSON/WebSocket API** — easier JSON/WebSocket source for JMA earthquake/tsunami info and community “I felt shaking” reports; free for commercial/non-commercial use, but unofficial relative to JMA. ([p2pquake.net](https://www.p2pquake.net/develop/json_api_v2/?utm_source=openai))  
3. **USGS Earthquake GeoJSON/FDSN API** — global earthquake data, good for cross-checking and nearby offshore events, but it won’t give Japan-specific **震度 / Shindo** the way JMA sources do. ([earthquake.usgs.gov](https://earthquake.usgs.gov/earthquakes/feed/v1.0/geojson.php))  
4. **Digital Typhoon** or **JMA typhoon XML** — for typhoon track/metadata around the western North Pacific. Digital Typhoon provides XML/JSON-style feeds including GeoJSON/Mf-JSON and CAP feeds, while JMA/RSMC Tokyo is the official typhoon authority for Japan-facing info. ([agora.ex.nii.ac.jp](https://agora.ex.nii.ac.jp/digital-typhoon/feed/index.html.en?utm_source=openai))  
5. **GDACS** — global multi-hazard impact alerts for earthquakes, tsunamis, tropical cyclones, floods, volcanoes, etc.; useful as a “global severity / impact” layer, not a substitute for local Japanese warnings. ([data.gdacs.org](https://data.gdacs.org/Knowledge/overview.aspx?utm_source=openai))  
6. **Open-Meteo JMA API** — not an alert API, but very handy for JSON forecasts using JMA models around Japan, e.g. wind gusts / precipitation thresholds. ([open-meteo.com](https://open-meteo.com/?utm_source=openai))  

For actual life-safety alerts, I’d still treat **JMA / local authorities / phone emergency alerts / NERV app** as the authority and any hobby app as supplementary. Tiny tail-flick of caution there.

## 1. JMA official XML feeds — the main one for Japan

JMA’s official developer portal publishes **気象庁防災情報XMLフォーマット** bulletins. It uses **Atom feeds**; you poll the feed, read entries, then fetch the linked XML bulletin files. JMA provides high-frequency feeds updated every minute with at least the most recent 10 minutes of messages, and long-term feeds updated hourly containing several days of messages. ([xml.kishou.go.jp](https://xml.kishou.go.jp/xmlpull.html))

The public XML list includes, among other things: typhoon information, weather emergency warnings/warnings/advisories, river flood forecasts, landslide warnings, record short-time heavy rain info, tornado advisories, heatstroke alerts, seismic intensity reports, earthquake source/intensity info, tsunami warnings/advisories/forecasts, tsunami observations, Nankai Trough earthquake info, volcano warnings/forecasts, ashfall forecasts, and eruption reports. ([xml.kishou.go.jp](https://xml.kishou.go.jp/xmllist.pdf))

### Useful JMA feed endpoints

```text
# High-frequency feeds: updated every minute, recent messages
https://www.data.jma.go.jp/developer/xml/feed/regular.xml   # 定時
https://www.data.jma.go.jp/developer/xml/feed/extra.xml     # 随時: warnings, typhoon, flood, landslide, tornado, etc.
https://www.data.jma.go.jp/developer/xml/feed/eqvol.xml     # 地震火山: earthquake, tsunami, volcano
https://www.data.jma.go.jp/developer/xml/feed/other.xml     # その他

# Longer feeds: updated hourly, several days of messages
https://www.data.jma.go.jp/developer/xml/feed/regular_l.xml
https://www.data.jma.go.jp/developer/xml/feed/extra_l.xml
https://www.data.jma.go.jp/developer/xml/feed/eqvol_l.xml
https://www.data.jma.go.jp/developer/xml/feed/other_l.xml
```

### Which JMA feed to poll?

For a Japan disaster dashboard:

```text
eqvol.xml   -> earthquakes, tsunami, volcano
extra.xml   -> typhoon, weather warnings/advisories, flood, landslide, tornado, heavy rain, heatstroke
regular.xml -> regular forecasts / outlooks, if needed
```

And periodically backfill from:

```text
eqvol_l.xml
extra_l.xml
```

That way, if your app sleeps or your server misses a short-lived feed entry, you can recover from the longer hourly feed. Also: cache downloaded XML files and do not refetch the same bulletin repeatedly; JMA explicitly says access causing 10 GB/day or more of downloads to the XML URLs can lead to IP blocking. ([xml.kishou.go.jp](https://xml.kishou.go.jp/xmlpull.html))

### Tiny Python sketch for JMA Atom polling

```python
import feedparser
import requests
from pathlib import Path

FEEDS = [
    "https://www.data.jma.go.jp/developer/xml/feed/eqvol.xml",
    "https://www.data.jma.go.jp/developer/xml/feed/extra.xml",
]

seen_path = Path("seen_jma_urls.txt")
seen = set(seen_path.read_text().splitlines()) if seen_path.exists() else set()

for feed_url in FEEDS:
    feed = feedparser.parse(feed_url)

    for entry in feed.entries:
        xml_url = entry.link

        if xml_url in seen:
            continue

        xml_text = requests.get(xml_url, timeout=10).text

        # TODO:
        # - parse XML
        # - inspect Control/Title, Head/Title, Head/ReportDateTime
        # - classify: earthquake / tsunami / typhoon / warning / etc.
        # - filter by area code / prefecture / municipality
        # - store notification state

        print(entry.title, xml_url)
        seen.add(xml_url)

seen_path.write_text("\n".join(sorted(seen)))
```

For production, I’d add ETag/Last-Modified support, backoff, XML schema-aware parsing, deduping by report datetime + serial number, and a persistent DB. Very glamorous, yes. My hoodie’s glowing lines are practically screaming “SQLite first, Postgres later.”

## 2. P2Pquake — easiest Japan earthquake JSON/WebSocket path

If you want something simpler than JMA XML for earthquake/tsunami, **P2Pquake** is probably the most developer-friendly Japan-specific choice. Their JSON API v2 provides JMA earthquake information, tsunami forecast lists/details, WebSocket support, and user “earthquake felt” reports; their docs state it is free for both commercial and non-commercial use with rate limits. ([p2pquake.net](https://www.p2pquake.net/develop/json_api_v2/?utm_source=openai))

Example endpoint shapes:

```text
# Earthquake list
https://api.p2pquake.net/v2/jma/quake

# Single earthquake detail
https://api.p2pquake.net/v2/jma/quake/{id}

# Tsunami forecast list
https://api.p2pquake.net/v2/jma/tsunami

# WebSocket endpoint is documented in their API spec
```

Use it for convenience, but for official wording / final authority, cross-check JMA.

## 3. USGS Earthquake API — good global cross-check

USGS provides real-time GeoJSON feeds and a flexible FDSN Event API. The GeoJSON feeds are intended as a programmatic interface and are updated every minute; the FDSN API supports custom searches by time, magnitude, bounding box, radius, etc. ([earthquake.usgs.gov](https://earthquake.usgs.gov/earthquakes/feed/v1.0/geojson.php))

### Japan bounding-box query example

```text
https://earthquake.usgs.gov/fdsnws/event/1/query?format=geojson&starttime=2026-06-25&minmagnitude=4&minlatitude=20&maxlatitude=50&minlongitude=120&maxlongitude=155&orderby=time
```

If you want a radius around Tokyo:

```text
https://earthquake.usgs.gov/fdsnws/event/1/query?format=geojson&latitude=35.6762&longitude=139.6503&maxradiuskm=1000&minmagnitude=4&orderby=time
```

USGS is especially useful for global/offshore situational awareness, but if you care about “what did people in Saitama/Tokyo/Chiba feel?” then JMA/P2Pquake-style Shindo data is more relevant.

## 4. Typhoon / hurricane sources

### For Japan / western North Pacific

Use **JMA** first. JMA’s RSMC Tokyo Typhoon Center provides tropical cyclone information for the western North Pacific and South China Sea, including present/forecast positions, movement, and intensity; JMA’s public Japan-facing tropical cyclone page is intended for people in Japan. ([jma.go.jp](https://www.jma.go.jp/jma/jma-eng/jma-center/rsmc-hp-pub-eg/RSMC_HP.htm?utm_source=openai))

For machine-friendly feeds, use:

```text
# JMA official XML “extra” feed includes typhoon bulletins
https://www.data.jma.go.jp/developer/xml/feed/extra.xml
https://www.data.jma.go.jp/developer/xml/feed/extra_l.xml
```

The public JMA XML bulletin list includes multiple typhoon-related products, including general typhoon info, position/development info, detailed position info, extended forecast-type typhoon messages, and probability of entering storm-wind areas. ([xml.kishou.go.jp](https://xml.kishou.go.jp/xmllist.pdf))

### Digital Typhoon

**Digital Typhoon** is very useful if you want typhoon-centric data and historical/search functions. It provides XML feeds such as Atom/RSS/Media RSS/CAP and JSON feeds such as GeoJSON and Mf-JSON, plus AMeDAS heavy rain / strong wind feeds. ([agora.ex.nii.ac.jp](https://agora.ex.nii.ac.jp/digital-typhoon/feed/index.html.en?utm_source=openai))

Digital Typhoon’s CAP feed is explicitly described as experimental/under testing, so I’d use it as a convenient data layer, not the final warning authority. ([agora.ex.nii.ac.jp](https://agora.ex.nii.ac.jp/digital-typhoon/cap/index.html.en))

### For Atlantic / Eastern Pacific hurricanes

Use **NOAA NHC** if you also care about hurricanes outside Japan’s basin. NHC provides RSS GIS feeds for Atlantic and Eastern Pacific basins and GIS datasets such as forecast track, cone of uncertainty, watches/warnings, wind fields, wind speed probabilities, and storm surge products. ([nhc.noaa.gov](https://www.nhc.noaa.gov/gis/rss.php?utm_source=openai))

NOAA also exposes ArcGIS-style web services where supported query formats include JSON and GeoJSON. ([mapservices.weather.noaa.gov](https://mapservices.weather.noaa.gov/tropical/rest/services/tropical/NHC_tropical_weather/MapServer?utm_source=openai))

But for Japan: **JMA / RSMC Tokyo > NHC**.

## 5. Other extreme weather / natural hazard APIs

### GDACS

GDACS is excellent for a global “what disasters are significant right now?” layer. It issues notifications for earthquakes and possible tsunamis, tropical cyclones, floods, and volcanoes, with automatic impact assessments for several hazards. ([data.gdacs.org](https://data.gdacs.org/Knowledge/overview.aspx?utm_source=openai))

Its quick-start says GDACS data are free and available through APIs, with examples for filtering earthquakes, tropical cyclones, floods, and alert levels; it asks users to acknowledge the source as “Global Disaster Alert and Coordination System, GDACS.” ([gdacs.org](https://www.gdacs.org/Documents/2025/GDACS_API_quickstart_v1.pdf))

Example shape:

```text
https://www.gdacs.org/gdacsapi/api/events/geteventlist/SEARCH?eventlist=EQ;TC&fromdate=2026-01-01&todate=2026-06-26&alertlevel=red;orange
```

Use GDACS for severity/impact context, not as a replacement for JMA warnings in Japan.

### NASA EONET

NASA EONET is an open API for continuously updated, near-real-time natural event metadata. It supports categories like severe storms and wildfires, returns event geometry, and can be queried as JSON/GeoJSON-style data. ([eonet.gsfc.nasa.gov](https://eonet.gsfc.nasa.gov/?utm_source=openai))

Example:

```text
# Open severe-storm events
https://eonet.gsfc.nasa.gov/api/v3/events?category=severeStorms&status=open

# GeoJSON-style events in a bounding box around Japan-ish region
https://eonet.gsfc.nasa.gov/api/v3/events/geojson?bbox=120,50,155,20&status=open
```

Again: nice for a map overlay, not an official local alert stream.

### Open-Meteo JMA API

Open-Meteo is useful if you want to build your own “notify me if wind gusts exceed X” or “heavy rain likely tomorrow” logic. It offers a free/no-key API for non-commercial use, returns JSON, and has a JMA endpoint optimized for Japan using JMA GSM/MSM models. ([open-meteo.com](https://open-meteo.com/?utm_source=openai))

Example for Tokyo wind/rain forecast:

```text
https://api.open-meteo.com/v1/jma?latitude=35.6762&longitude=139.6503&hourly=precipitation,wind_speed_10m,wind_gusts_10m&daily=precipitation_sum,wind_gusts_10m_max&timezone=Asia%2FTokyo
```

This is forecast/model data, not an official warning. Still useful for personal dashboards.

Here’s the clean setup I’d recommend:

```text
                 ┌──────────────────────┐
                 │  JMA XML feeds        │
                 │  extra.xml / eqvol.xml│
                 └──────────┬───────────┘
                            │
                            ▼
┌───────────────┐     ┌──────────────┐     ┌────────────────┐
│ P2Pquake WS   │ --> │ Ingestion DB │ --> │ Alert engine    │
│ JSON quake    │     │ events table │     │ dedupe/filter   │
└───────────────┘     └──────┬───────┘     └───────┬────────┘
                             │                     │
┌───────────────┐            │                     ▼
│ USGS / EMSC   │ ---------- │              ┌─────────────┐
│ cross-check   │            │              │ Notifications│
└───────────────┘            │              │ map/web/push │
                             │              └─────────────┘
┌───────────────┐            │
│ GDACS / EONET │ ---------- │
│ global layer  │            │
└───────────────┘            │
                             ▼
                      ┌──────────────┐
                      │ Map renderer │
                      │ Leaflet etc. │
                      └──────────────┘
```

### Suggested polling strategy

| Task | Source | Frequency |
|---|---|---:|
| Earthquake/tsunami official bulletins | JMA `eqvol.xml` | Every 60s |
| Weather/typhoon warnings | JMA `extra.xml` | Every 60s |
| Missed message recovery | JMA `eqvol_l.xml`, `extra_l.xml` | Every 10–30 min or hourly |
| Friendly earthquake JSON / WebSocket | P2Pquake | WebSocket or gentle polling |
| Global earthquake cross-check | USGS / EMSC | Every 1–5 min |
| Global impact layer | GDACS | Every 5–15 min |
| Forecast thresholds | Open-Meteo JMA | Every 1–3 hours |

### Data model idea

```sql
CREATE TABLE hazard_events (
    id TEXT PRIMARY KEY,
    source TEXT NOT NULL,
    hazard_type TEXT NOT NULL,  -- earthquake, tsunami, typhoon, heavy_rain, flood, volcano, etc.
    title TEXT,
    issued_at TIMESTAMP,
    updated_at TIMESTAMP,
    area_name TEXT,
    area_code TEXT,
    severity TEXT,
    lat REAL,
    lon REAL,
    raw_url TEXT,
    raw_payload TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

For Japan, I’d strongly recommend keeping the **raw XML/JSON** in storage. JMA bulletins can be nuanced, and you’ll be grateful later when you need to re-parse or fix a parser bug. Ask me how my tail knows.

## Specific notes for Japan

### Earthquake

Use:

```text
JMA eqvol.xml        -> official earthquake/tsunami/volcano bulletins
P2Pquake             -> convenient JSON/WebSocket, JMA quake/tsunami, user felt reports
USGS / EMSC          -> global cross-check
```

If your UI is for people in Japan, display **震度 / Shindo** prominently, not just magnitude. Magnitude tells energy at the source; Shindo tells how strong shaking was at locations.

### Tsunami

Use:

```text
JMA eqvol.xml        -> official Japan tsunami warnings/advisories/forecasts
P2Pquake /jma/tsunami -> convenient JSON list/detail
GDACS                -> global possible-tsunami impact context
```

For any user-facing tsunami alert, link/point users to JMA/local authority instructions and avoid over-interpreting.

### Typhoon

Use:

```text
JMA extra.xml        -> official typhoon bulletins for Japan
JMA/RSMC Tokyo       -> official WNP/South China Sea tropical cyclone info
Digital Typhoon      -> convenient typhoon feeds, historical data, GeoJSON/Mf-JSON
GDACS                -> global impact/severity layer
Open-Meteo JMA       -> wind/rain forecast thresholds
```

### Heavy rain / flood / landslide / tornado / heat

Use:

```text
JMA extra.xml        -> official warnings/advisories and hazard bulletins
JMA AMeDAS CSV/data  -> observed rain/wind/temperature etc., if needed
Digital Typhoon AMeDAS feeds -> heavy rain / strong wind feed convenience
Open-Meteo JMA       -> forecast thresholds
```

JMA’s developer portal also says current AMeDAS observation data such as precipitation, max/min temperature, max wind speed, and snow depth are available as machine-readable CSV downloads. ([data.jma.go.jp](https://www.data.jma.go.jp/developer/))
